Accessibility Statement for Historical Texts

This accessibility statement applies to https://historicaltexts.jisc.ac.uk and separate statements will be produced for the UK Medical Heritage labs site.

This website is run by Jisc. We want as many people as possible to be able to use this website. For example, that means you should be able to:

  • Change colours, contrast levels and fonts using browser plugins
  • Zoom in up to 200% without the text spilling off the screen.
  • Navigate most of the website using just a keyboard
  • Listen to most of the website using a screen reader (tested using NVDA)
  • Skip to main content on all but one page
  • Access audiodescription for videos without sound

We’ve also made the website text as simple as possible to understand.

AbilityNet has advice on making your device easier to use if you have a disability.

How accessible this website is

We know some parts of this website are not fully accessible:

  • Although we have made recent improvements to the site, the viewer still has some issues. If you are having difficulties, the best solution may be to open the item in the viewer and download the PDF/text files from the download menu. This can be operated by keyboard and, even when styling is turned off for the viewer page, the buttons still operate. This would also be our recommendation if you need to use the viewer at a magnification above 200%.
  • The site is not fully usable in the portrait orientation on a small screen device. We recommend viewing the site in landscape orientation if possible. If this is not possible, we recommend viewing search results in the portrait orientation and then downloading the PDF from the viewer rather than trying to operate the viewer in portrait orientation.
  • When you view some of our pages at or above 200% zoom, although the text reflows, parts of some images and videos are lost off the right-hand side of the page. Some of these images can be viewed as full page images (e.g. in the Quick Reference Guide) so check if they can be clicked on to enlarge them that way. An alternative source for our support videos is the Historical Texts YouTube channel.
  • Although we have worked to make our newest PDFs accessible (checking them for compliance with Adobe Acrobat DC) we recommend using the HTML version of our Quick Reference Guide for those using assistive technology and the more accessible .doc version of the Quick Start Leaflet on the librarians page.
  • It is not possible to use the histogram to narrow date ranges with a keyboard. It is possible to achieve the same result using the advanced search date filter, which can be operated by keyboard.

Feedback and contact information

If you need information on this website in a different format like accessible PDF, large print, easy read, audio recording or braille:

  • email historicaltexts@jisc.ac.uk

We'll consider your request and get back to you.

Reporting accessibility problems with this website

We're always looking to improve the accessibility of this website. If you find any problems not listed on this page or think we’re not meeting accessibility requirements, contact the helpdesk (historicaltexts@jisc.ac.uk).

Enforcement procedure

The Equality and Human Rights Commission (EHRC) is responsible for enforcing the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018 (the 'accessibility regulations'). If you’re not happy with how we respond to your complaint, contact the Equality Advisory and Support Service (EASS).

Technical information about this website's accessibility

Jisc is committed to making its website accessible, in accordance with the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018.

Compliance status

This website is partially compliant with the Web Content Accessibility Guidelines version 2.1 AA standard, due to the non-compliances and exemptions listed below.

Non-accessible content

The content listed below is non-accessible for the following reasons.

Non-compliance with the accessibility regulations

A number of our pages lack headings which makes it harder for those using screen readers to navigate. This fails WCAG 2.1. success criterion 1.3.1 (Info and Relationships). There is also a drop-down autofill for tags in the viewer which lacks a description which fails the same criterion. We plan to ensure that all pages in the new site we are designing have appropriately used headings and descriptions on all pages.

If styling is removed from the viewer page, the content is no longer fully usable. This fails WCAG 2.1 success criterion 1.3.2 (Meaningful sequence). The best solution to this is to download the PDF/text and read it outside the viewer. This operation can still be carried out from the page with the styling turned off.

The site was originally designed with only desktop use in mind, and although improvements have been made to increase the usability on smaller screen tablet/mobile devices, it is still not really possible to use the site properly in a portrait orientation. This fails WCAG 2.1 success criterion 1.3.4 (Orientation). We plan to ensure that the new site we are designing is usable in both portrait and landscape orientation from the start.

There are a small number of pages that still have text and graphical object contrast issues which fail WCAG 2.1 criteria 1.4.3. (Contrast Minimum) and 1.4.11. (Non text contrast). We will ensure the issues are eliminated on the new platform we are developing.

When the site is zoomed in to 200% or more, although text reflows as required, some images and videos "fall off" the right-hand side of the page rather than resizing. When using the viewer at a magnification of above 200% it becomes very difficult to use the content of the side panels and some of this is lost. These fail WCAG criterion 1.4.10 (Reflow). Some workarounds for this issue are listed earlier in this document. We plan to use a fully fluid layout for the new site we are designing which will eliminate these issues.

Please note that the zoom in and zoom out keys in the viewer cannot be operated by keyboard when using the viewer in Chrome which fails WCAG 2.1 criterion 2.1.1 (Keyboard). A workaround for this is to use the viewer in FireFox which does not suffer from this issue, caused by a bug in our third party viewer. It is also not possible to operate the histogram using the keyboard but the same result can be achieved using the advanced search date filter, which can be operated by keyboard. It is not currently possible to operate the fuzzy searching option using the keyboard. This is the result of a bug which we are working to fix. This statement will be updated when the bug is fixed.

A "Skip to main content," link is not currently available in the viewer which fails WCAG 2.1 criterion 2.4.1. (Bypass Blocks). These have been added to all other pages, but technical difficulties prevent it from being added to the third party viewer page. We plan to ensure that all pages in the new platform have bypass blocks in place.

Most pages have a unique title but currently the search results page and browse pages have the identical title of "results." This makes distinguishing these pages for those using a screen reader more difficult and fails WCAG 2.1 criterion 2.4.2 (Page titled). We plan to ensure that all pages have unique titles in the new platform we are building.

On some pages, there are links with the same text that go to different places. It is also possible to have multiple saved searches with the same name. This leads to a list of links with the same names but going to different places. This can be confusing for screen reader users and fails WCAG 2.1 criterion 2.4.4 (Link Purpose In Context). We plan to differentiate these links where possible either by updating the text or using aria-labelledby by 23 September 2020. For those where this is not possible due to the structure of the site, we will change we way we create menus in the new site we are designing. We also plan to change the way that the saved searches operate in our new platform.

We are aware that although improvements have been made to the navigability of the site by keyboard/screen reader, it is still sometimes difficult to see where you are which fails WCAG 2.1 criterion 2.4.7 (Focus visible). We plan to use a design pattern which will make focus states more consistently perceivable in the new site we are designing.

Currently it is possible to enter incorrect text into the date field of the advanced search and no error message is generated. This fails WCAG 2.1 criterion 3.3.1 (Error Identification). As we build our new platform, we plan to have a consistent approach to error reporting and this will be resolved.

Our viewer uses a form to render the content but lacks a submit button. This fails automated tests for WCAG 2.1 criterion 3.3.2 (Labels or instructions). This can only be fixed using a new viewer in the platform we are developing.

Disproportionate burden

We are not claiming a disproportionate burden for any area of improvements that are required.

Content that's not within the scope of the accessibility regulations

PDFs and other documents

A great deal of effort has been undertaken to make any newer PDFs on our site as accessible as possible. We are aware that there are some older PDFs on the site (mainly regarding the old platform) which may not be accessible.

The accessibility regulations do not require us to fix PDFs or other documents published before 23 September 2018 if they’re not essential to providing our services.

Any new PDFs or Word documents we publish will meet accessibility standards or the information will be provided in a more accessible HTML format.

What we're doing to improve accessibility

We plan to bring the design and maintainence of the Historical Texts site in house over the next two years, including a package of work on redesigning some aspects of the site and moving to the use of the Universal Viewer. As part of this work we will be bulding the site with accessibility as a foundational principle.

Preparation of this accessibility statement

This statement was prepared on 8 September 2020. It was last reviewed on 30 April 2021.

This website was last tested on 15 April 2021. The test was carried out members of the Historical Texts team.

We selected several pages that represented different types of content on the site ensuring there were images, links and forms across the selection. We tested:

  • Historical Texts home page
  • Search results page
  • Viewer
  • Support home page
  • Quick Reference guide page
  • About home page
  • Collections
  • Browse page
  • Personalisation page

We used a mixture of manual testing (including using NVDA) and automated testing using both WAVE and SiteImprove plugins.
