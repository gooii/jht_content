Help file

This is the Help page for the Burney Newspaper Collection. It is intended to assist researchers using the collection via this Historical Texts labs site.

Navigation bar

There is a black horizontal navigation bar at the top-right of the screen on every page of Historical Texts so you can quickly navigate to any part of the website from any page you are on. Some options are only be available on certain pages though, for example the Download button is only enabled when you have opened a publication in the viewer.

Features of the Burney Collection

FULL TEXT - Full text is available for every title in the collection. The text is machine generated using Optical Character Recognition (OCR) software.

PDF DOWNLOADS - PDFs are available for every title, and the PDF for a whole publication can be downloaded directly using a URL link.

PAGE IMAGE DOWNLOADS - Images for every page are available and can be downloaded individually in the JPEG format.

OTHER DOWNLOAD OPTIONS - Currently the bibliographic citation information cannot be downloaded.

Searching

We are using Elasticsearch open source software to index the data in Historical Texts.

To run a search, enter your search terms in the Search box and select the "Search" button. By default, this will search all of the information in the bibliographic records as well as the full text. The search software is not case-sensitive so "Shakespeare" will return the same results as "shakespeare". The number of matching results will be shown to the right of the Search box.

It is worth noting that you can see where in a publication your search terms match using the "Pages" tab in the "Search" panel, within the Viewer - and that you can then search for new terms within the publication using the Search box at the top of the "Pages" tab. For more information, see the section on the viewer.

If you do want to limit your search to a particular field e.g. Title you can do so using the drop-down list to the left of the Search box.

The following options are available:

  • All
  • Title
  • Bibliographic number
  • Genre
  • Text type

Building up complex queries

To build up a complex query you can either type your query into the search box directly using boolean operators or use the add another row feature. Simply select "Advanced Search" on the search page to show additional search features. From here you can add new rows and specify whether your terms MUST, SHOULD or MUST NOT appear in your results.

The first clause of the search (in the main search box) is a MUST - so the results must include whatever goes in that first box. This means that whatever else goes in the search you are essentially starting from a situation where the search in the first box is compulsory. This means that adding a SHOULD clause to this cannot increase the set of results. Adding a SHOULD clause cannot reduce the set of results either (as the clause is optional).

This means that adding a SHOULD clause does not change the total number of results. What it does do is re-order the results when using the relevance ranking, so those items matching the SHOULD clauses come to the top.

Operators, Regular Expressions and Reserved Characters

Boolean operators

The familiar Boolean operators AND, OR, NOT (alternatively you can write them as &&, ||, !) are supported and can be entered directly into the search box. NOT takes precedence over AND, which takes precedence over OR and you can combine these operators with brackets. These operators can affect the terms to the left and right of them.

You can also use the preferred Elasticsearch operators ‘+’ (which means the term MUST be present) and ‘-’ (which means the term MUST NOT be present). These operators can only affect the terms to the right of them.

The example below finds all results for "Shakespeare" without "Venice" in.

                Shakespeare -Venice

By default, any terms you enter in the search box MUST be present. For instance, all of the searches below are identical:

           	Shakespeare -Venice ode
			
            	+Shakespeare -Venice ode
			
            	+Shakespeare -Venice +ode

Wild card operators

Two wild card operators are available:

                * = Match any character, including no character
                ? = Match any single character; will not match no character

Please avoid wild card characters occurring at or before the third position as they may produce too many unrelated matches.

Proximity operators

Proximity operators enable you to specify the distance (in number of words) your search terms are from each other in the text. You can specify the number using a tilda and a number after the term itself, to widen the proximity increase the number.

The example below finds king and queen within 5 words or less of each other.

                "king queen"~5

Regular expressions

Searching using regular expressions is supported and terms can be entered directly into the search box wrapped in forward slashes "/". Historical Texts uses Elasticsearch software to index the data and the full list of supported operators for regular expressions can be found on the Elasticsearch website.

The example below shows a regular expression to look for the word William.

                /W[ilam]{0,6}/

Regular expressions support

Support information and tutorials to help you construct regular expressions can be found in a range of places on the web, for example at the Regex One site and Regular Expressions Info.

Reserved characters

Certain characters function as operators in the system and if you want to use them in your search query, but not as an operator, then you will need to escape them with a leading backslash.

The reserved characters are:

                + - && || ! ( ) { } [ ] ^ " ~ * ? : \ /

Fuzzy searching

Fuzzy searching enables you to search for terms that are similar to, but not exactly like our search terms.

"Fuzziness" is measured in terms of the number of edits (insertions and deletions) between one term and another. You can specify the number of edits using a tilda and a number after the term itself.

The example below finds "king", and all other terms within a single edit of the word "king":

king~1

The maximum "fuzziness" supported is 2.
(obviously you can use the "Fuzzy search" checkbox for fuzzy searching but the operator is also supported in the main search box and allows you to apply the "fuzziness" to only certain terms within your search)

Variant Forms and Misspelling searching

Users can also choose options for "Variant Forms", "Variant Spellings", "Misspellings" and "Printer Replacements" by clicking the "Fuzzy and Variants" drop-down button and ticking the desired options.
All the Variant words that are added to the search will appear below the Search bar, colour-coded by Variant type. You can remove Variant words from your search by clicking the 'x' at the end of the icon.

We recommend that users not use both the 'Fuzzy' and 'Variants' options at the same time, as this may yield unpredictable search results.

Grouping

You can group together terms or search clauses together using parentheses. For example:

Shakespeare AND ((merchant AND -Venice) OR (king AND lear))

Field searching

You can search specific fields by using the field name followed by a colon then the search term. For example

title_all_text:Shakespeare

You can use this in combination with other operators. For example:

title_all_text:(Shakespeare lear)
title_all_text:lear AND author_all_text:Shakespeare~2

You can also search for the presence/absence of fields using '_missing_' and '_exists_'. For example to find all texts without edition information:

_missing_:publication.edition.statement

Bibliographic fields supported in the drop-down Search box menu:

  • All: _all
  • Title: title_all_text
  • Bibliographic number: bibliographic_number
  • Genre/Material type: type_or_genre_all_text

Also dates:

  • Earliest publication date for text: lifecycle.publication.date.earliest
  • Latest publication date for text: lifecycle.publication.date.latest

Ranges (numeric, date or text)

You can search for numeric, date or text ranges. Inclusive ranges are specified with square brackets [min TO max] and exclusive ranges with curly brackets {min TO max}.

lifecycle.publication.publisher.name.value_text:[aa TO ab]

The example above would find all texts with a word in the publisher that comes between "aa" and "ab" in alphabetical order including occurrences of 'aa' and 'ab'. For example it would find the publication statement:

Aat the shop adioyning vnto Saint Mildreds Church in the Pultrye by John Allde

Because of the word "Aat"

lifecycle.publication.date.earliest:[* TO 1600}

Finds all texts with an earliest publication date before 1600 (up to 1599).

More details on range searching can be found on the Elasticsearch website.

Boosting

Boosting can be used to change the relevance ranking based on specific terms in your search - it won't change the results retrieved, but will change the order. This is done by using the boost operator ^. For example:

Shakespeare lear^2

Will boost the importance of 'lear' in the results and so re-order the search set to emphasize those where 'lear' appears more often.

The default boost value is 1, but can be any positive number. A boost value between 0 and 1 reduces relevance of a term.

Advanced search

More search options are available, but are hidden by default. To show the extra options, click the "Advanced Search" drop-down button to the right of the page.

The extra options are:

  • Filter by date - included undated, exclude undated, undated only, or specify a date or range
  • Search on Description only, Full Text Only, or both
  • Add rows to introduce additional search terms, using MUST, SHOULD or MUST NOT conditional logic, and searching on any fields available in the normal Search box
  • Reset button to clear down the search interface

Filter by date

By default, searches will return data from all available date ranges. Clicking the drop-down arrow and selecting the radio button next to the date range field allows users to select from "Including Undated", "Excluding Undated" or "Undated Only" options.

"Including Undated" is the default for searching. It also allows for undated records to be included alongside dated records matching a specified range as entered into the date range field (e.g. 1600-1645 plus any undated records matching the other search terms).

"Excluding Undated" will return only dated records matching a specified range as entered into the field (e.g. 1600-1645).

"Undated only" will return only records with no date

Please note that the "Including Undated", "Excluding Undated" and "Undated only" Date Filter options are not available when the Periodicals button is selected.

Individual years can be entered into the date range field, as can hyphenated ranges (e.g. 1600-1645).

Selecting specific collections in the avanced search options may produce 0 results since only the Burney data is avaible in the labs site. Users can also use the Selector tool on the Histogram to further refine their search results by date range.

Specify which collection or collections to search on

Although the interface allows the selection of different collections, these are not included in the labs site. By default your search will be conducted over the Burney Collection only. Selecting a different collection in the advanced search options may produce 0 results as those collections are not included in the labs data.

Search on Description only, Full Text Only, or both

By default, Historical Texts searches all full text and bibliographical data available. To restrict your search to only the bibliographic data (and not the full text) of the records, select the "Description" radio button. To search only the full text (and not the bibliographic data) select the "Full Text" radio button.

Using the "Full Text" button will also ensure that only results with the full text included will be returned.

Filter by Illustrated, Not Illustrated, or both

To restrict your search to records specifically with or without illustrations, expand the Advanced Search options and click the "Illustrated" or "Not Illustrated" radio button as required.

Add rows to introduce additional search terms

To build up a complex query you can either type your query into the search box directly using Boolean operators or use the "New Row" feature. Simply select "Advanced Search" on the search page to show additional search features. From here you can add new rows by clicking the "New Row" button, and specify whether your terms MUST, SHOULD or MUST NOT appear in your results.

Reset button to clear down the search interface

Should you wish to reset the search interface back to the default values, you can click the red "Reset" button at the bottom of the "Advanced Search" options should you wish. Manually setting the values back to their defaults should work equally well.

Browsing

The browse options in the labs site are designed to browse the standard Historical Texts collections. They browse by author and printer/publisher, two pieces of metadata that are not present in the Burney Newspaper collection.



Please note that the Thomason Tracts data is not accessible in the Burney labs site. To access this, please visit the main Historical Texts site.

Results

Histogram

Once you've run a search on Historical Texts, a Histogram displaying the date range distribution of your results graphically will be shown above your results list. Number of hits is shown on the Y-axis and time in years is shown on the X-axis. If you don't wish to see the Histogram, you can hide it by clicking the upwards-acing arrow on the Date Range tab.



You can click and drag within the Histogram to make a further date range selection within your existing results. When you release the mouse, the number of results will be previewed by the scissor icon. Click the scissor icon to confirm the date selection.

Please note, it is possible to make searches, or date range selections within a search that make the Histogram show a solid block of colour. While it might seem confusing, this is not incorrect - it's just a reflection of the how the results data has been filtered down.

List view - Grid view

Should you wish, you can toggle between a list view or a grid view of your search results by clicking the List or Grid icon buttons, shown next to the "View Mode" legend. When in Grid view, more results are visible on the screen at any one time, so less bibliographic data for each record is displayed by default. When in either view, click "More Information" to see the full bibliographic data for any given result, then "Less Information" to hide it again.

Viewing search term matches within a text

Beneath each search result on the results page is a "matches within text" link. Click on this to display pages where your search terms match within the publication. If "Showing 0 matching image(s) out of 0" is displayed instead then this may be due to the matches being in the bibliographic information, for example the title or author, instead of the full text.



You can click on a thumbnail or image number to go directly to that page of the text in the viewer. You can also select the thumbnail or title for the publication to open the whole publication in the viewer and select the left hand "Search" panel, then the "Pages" tab to view all of the search term matches as well.

Burney Periodicals display

Publications that are part of a series are displayed differently, to enable you to quickly identify them and navigate the individual issue you require. If you search for a particular periodical e.g. ‘Mercurius Politicus’, you'll see that there is a bibliographic record for the periodical which has a ‘Series’ thumbnail. Clicking this thumbnail transfers users to the ‘Periodicals’ tab on the results page where the list of individual issues will be displayed. You can then select an issue to open in the viewer as usual.

Individual issues of periodicals are displayed on the ‘Periodicals’ tab in issue order.

Sorting and ordering results

You can order your search results by Relevance, Date (Ascending), Date (Descending), Author (A-Z), Title (A-Z). The default value is Relevance.

Filtering results

You can filter down the results of an existing search using the Printer/Publisher, Author, Date, Subject, Text Type and Genre filters on the left hand side of the screen. Note that since only one collection is present in the labs site, the use of the collections filter will have no effect on your results.

Number of results displayed

You can also change the number of results shown per page using the drop-down arrow under the histogram, and navigate through your pages of results with the results navigation toolbar shown at the bottom of each page. You may want to hide the thumbnails on the results page if you are viewing 90 or above results per page as the page will take longer to load with the thumbnails. See the section below on how to do this.

Show and hide results page thumbnails

Results page thumbnails are displayed next to each search result on the results page by default. If you don't want to see them you can use the Thumbnails buttons above the filters on the left hand side of the results page to hide them.

You may wish to do this if you are choosing to view large numbers of results per page e.g. 90 or above, as the page will take longer to load with the thumbnails.

Please note: this setting only applies to viewing thumbnails on the results page, not the thumbnails in the left hand panel in the viewer for each text.

Viewer

You can view a screencast about the Viewer and its features here:

To view the page images for a text simply select a record from your results list or the browse pages by clicking on the thumbnail or title hyperlink for it. This will open up the viewer, enabling you to view images, full text (where available) and thumbnails side by side. Using the navigation bar at the top of the screen, you can also download individual images (JPEGs) and whole publications (PDFs) and bibliographic citation information (RIS) and get a permanent URL for the publication when viewing an image.

The viewer is comprised of three panels: a central panel displaying the images and a search panel to the left and a details panel to the right which are collapsed by default. To open the left hand panel select the "Search" button on the left hand side of the viewer and to open the right hand panel select the "Details" button on the right of the viewer as shown in the screenshot below.



You can have all three panels open at the same time, or just one or two of them and you can re-size the panels by dragging the handle in the middle of the panel edge (highlighted by orange boxes in the screenshot below) to the left or right. To close either panel click on the cross at the top of the panel edges, highlighted by the small orange boxes in the screenshot below.



The options available on each of the three panels are listed below.

Image panel

The central panel in the viewer displays the page images. There is a toolbar on the left hand side of the panel providing the following options:

  • Zoom in
  • Reset zoom (reset the zoom level so the image is the original default size)
  • Zoom out
  • Full screen (opens the page image in full screen mode) - please note the Search and Details panels are not available when in full screen view, as this is intended to allow as much screen area to the images as possible.
  • Highlight off (feature not available yet but it will hide the search term highlighting)
  • Rotate (rotate image by 90 degrees at a time)

You can use your mouse wheel or trackpad to zoom in and out instead of using the zoom in/out buttons if you prefer and you can drag the image around using your mouse or trackpad.

You can navigate through the publication using the navigation toolbar to the bottom of the screen. Here you can move forwards or backwards by page, jump to the start or end of a text, toggle between a single page or two pages reading view, or type a page number in the box and jump directly to that page. These will stay visible even if you switch to the full screen view using the left hand toolbar.

Search panel

To open the search panel, select the "Search" button on the left had side of the viewer. You can re-size the panel by dragging the handle in the middle of the panel edge across to the right and you can close the panel using the cross at the top of the panel edge. There are three tabs providing options for navigating publications:

Pages tab

Pages - this lists images matching your search terms so you can select one to jump straight to that image. You can also enter new search terms into the box on the Pages tab to search within the text itself, then click any of the results below to jump to the relevant page in the Image Panel. Note that the success of this may be dependent on the quality of the OCR text. You can order these results in either "Pages Ascending" or "Page Descending" using the drop-down arrow as required as well as select to search for variants and or illustrated pages using the drop down box beneath the search box.

Thumbnails tab

This tab displays thumbnail images for every image in the publication. 30 are loaded at a time and as you scroll down additional thumbnails will continue to load until you reach the end of the publication. Simply click on a thumbnail image to jump directly to that image in the central viewer panel.

Publications tab

The publications tab displays all of the other publications in Historical Texts that match your search terms, so you can quickly jump to a different publication from within the viewer instead of returning to the results page to select another text.

Details panel

To open the details panel click on the "details" button on the right hand side of the viewer. You can re-size the panel by dragging the handle in the middle of the panel edge across to the left and you can close the panel using the cross at the top of the panel edge. There are four buttons providing additional information on the publication: details, contents, text and tags.

Details

The details button displays the bibliographic information for a publication.

Contents

The table of contents feature is active but improvements will be made to it in due course.

Text

The text button displays the full text supporting the images, where available. You can also navigate through the publication using the navigation toolbar to the top of this tab. Here you can move forwards or backwards by page, jump to the start or end of a text, toggle between a single page or two pages reading view. This might be preferable for users who primarily want to read from the full text rather than the images.

Please note that full text is not available for all publications in Historical Texts. Publications with full text present will display a "T" icon in a grey square under the "details" button when the panel is minimized. When full text is not available the full text tab will display the message "Full text not available".

Tags

The tags button enables you to create and view tags for individual pages within the publication you are viewing. To create a tag open the page you want tag in the viewer and enter a name for the tag in the box at the top of the tags screen and click on the "add tag" button. The tag will be displayed below, along with any other tags you have created in the publication. Click on the page number for any tag to jump directly to that page.

To view all of your tags for all publications select the "My Texts" then "My Tags" option from the black navigation bar at the top of the screen.

Please note: The My Tags feature requires logging in with an institutional account via the UK Federation (Shibboleth and OpenAthens) and is not available if you have accessed Historical Texts via IP authentication.

Downloading

When you have opened a publication in the viewer, choose the "Download" button and select from a range of download options. The following options are available:

  • individual image currently open in the viewer as a JPEG file
  • whole publication as a PDF file
  • citation information in .RIS format to import into reference management software

Please note: the Download button is only enabled on the Navigation bar when you have opened a publication in the viewer. On all other pages it is currently disabled and greyed out with a message stating "feature not available" as you can only download a publication when you have selected and opened it.

Stable URLs

The "Share" button on the Navigation bar at the top of the screen provides a stable (permanent) URL for the publication you are viewing for you to copy and paste into your documents or web pages. The button is deactivated and greyed out until you open a publication in the Viewer, at which point it is enabled. When you select the button a stable URL for the publication will copy to the clipboard.

 


Saving searches and publications

You can save searches and publications for future sessions in Historical Texts.

  • Save searches: A green "Save Search" button enables you to save your current search. Your saved search can be viewed and run from the "saved searches" option under "My Texts" on the toolbar at the top of the screen.
  • Save publications: You can save publications to a list for future reference by selecting the checkbox next to one or more results from your results list. The publications will be added to the "My Saved Items" tab and from there you can add them to a new list or an existing list. You can also access your saved items from "My Texts" on the toolbar at the top of the screen.

To access previously saved searches and lists, click the "My Texts" drop-down button, and select either "Saved Searches" or "My Saved Items".

You can create up to 99 searches and lists as well as save up to 99 publications in each list.

"Saved Searches" will show any previously saved searches in a list, as well as the date of creation and how many results it returned. You can click on the "Edit" or "Delete" buttons to the right to update or remove them as required. Clicking on the name of a search will cause it to be re-run. Please be aware that the data and indexes in Historical Texts are potentially subject to updates, which may cause previous searches to return differing results when run anew.

"My Saved Items" will display any collections you have previously made. You can make new ones by clicking the green "Create New Collection" button, and name it by clicking the blue "Edit" button and typing in the name field. Once editing, additional buttons for "Save" and "Cancel" will appear. Unsaved changes will not be applied. You can delete collections if you need by clicking the red "Delete" button to the right of the page.

Personalisation options are only available when logging in via an institutional account (UK Federation Shibboleth/OpenAthens). If you access Historical Texts directly (i.e. using IP authentication) then you will be prompted to login with an institutional account when you click on "save search", "My Saved Items" or an option under "My Texts".

Setting default search and display preferences

You can set default search preferences for future sessions if you access the service using an institutional account (Shibboleth/OpenAthens) rather than IP authentication. To do this select the Settings Cog icon on the Navigation bar.



The options are:

  • Thumbnails on the results page: On/Off
  • Advanced search panel: Open/Closed
  • Select collection to search: this feature should not be used in the labs site as only the Burney Collection data is included

Simply choose the options you want and click on the close button. If you experience any problems using these options then try the "Reset" button to reset your preferences back to the default settings and try again.

How to seek permission to reproduce images from Burney

The Burney Newspaper Collection data in this labs site has been provided by Cengage. The copyright for all images is owned by the publishers and so permission to reproduce them must be sought from them.

You do not need permission to use images in unpublished works such as theses unless they will be available on the open web or in an institutional repository or other database. To request permission to reproduce a work or to seek more information, contact the publishers directly:

Cengage have an online request form for this purpose. A link to the form can be found on the Cengage Rights and Permissions page
