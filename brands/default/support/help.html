Navigation bar

There is a black horizontal navigation bar at the top of the screen on every page of Historical Texts so you can quickly navigate to any part of the website from any page you are on. Some options are only be available on certain pages though, for example the download and share buttons are only enabled when you have opened a publication. Other options such as "My Texts", "Print" and "Settings" are still in development and will be coming soon, so these features are currently unavailable.

Searching

Standard search

We are using Elasticsearch open source software to index the data in Historical Texts.

To run a standard search enter your search terms in the search box and select the "search" button. By default, this will search all of the information in the bibliographic records as well as the full text and you can enter bibliograhic numbers (ESTC number for ECCO, STC or Wing number for EEBO) into the search box when searching over all fields not just the bibliographic number field. The search software is not case sensitive so "Shakespeare" will return the same results as "shakespeare". The number of matching results will be shown to the right of the search box.

If you do want to limit your search to a particular field e.g. title, author or bibliographic number you can do so using the drop down list to the left of the search box.

The options available are:

  • All
  • Title
  • Author
  • Printer/Publisher
  • Place of publication
  • Bibliographic number
  • Language
  • Illustration description
  • Subject
  • Genre

Author

You can enter the author name in either first name, last name or last name, first name format and it will find the same results. 

                Thomas, Moore
                Moore, Thomas

Bibliographic number (STC, Wing and ESTC numbers)

Records in EEBO and ECCO have bibliographic numbers and you can use these to search for specific records in the system. For EEBO there are STC, Wing and Thomason Tract numbers. For ECCO, there are ESTC numbers. The British Library 19th century collection does not have a bibliographic reference number.

You can enter bibliographic numbers directly into the default "all" fields search as well as in the specific bibliographic number search.

Please note that you will need to be quite exact when using the bibliographic reference number search, for instance whether you enter a full stop or not along with the identifier will affect the results returned. This issue can be worked around by either using the 'All' fields search and just putting the bibliographic number in there (but sometimes this can result in lots of hits), or by using a wildcard character (*) at the end of the number.

Collection Bibliographic number Example Comments
EEBO STC, Wing and Thomason STC 21805.2
D489AB (Wing 2nd ed.)
669.f.8[43]. (Thomason)
 You do not need to enter "STC" or "Wing" as well as the number, but it may help narrow down your search if there are mulitple records which contain part of the same number.
ECCO ESTC number T067061 ESTC numbers sometimes have leading zeros. They are listed without the leading zero on the ESTC site but with them in the bibliographic information provided to Historical Texts, so when searching in Historical Texts you need to include the leading zero(s), so the format is a letter followed by six numbers.

Genre/material type

Genre/material type describes both the form of the item and the content in EEBO. We have merged Genre/material type because in the metadata the physical form and content type are often mixed up and the number of terms are fairly small. The material type in some cases is also treated as a subject term. There are 6 broad types, 21 sub-types (which is a mixture of more specific material type and 'content' type), and over 300 form and genres.

Collection Type Sub-type Form and Genre
EEBO There are 6 broad types: Book, Periodical, Music, Newspaper, Map and Continuing Resource. There are 21 sub-types including Bibliography, Dictionary, Full score, Legislation... There are over 300 of these including Broadsides, Early works to 1800, Poetry, Sermons, Title pages, Selections, Maps, Map (Single map), Commentaries, Controversial literature, Terminology, Sources...

Building up complex queries

To build up a complex query you can either type your query into the search box directly using boolean operators or use the add another row feature. Simply select "Advanced Search" on the search page to show additional search features. From here you can add new rows and specify whether your terms MUST, SHOULD or MUST NOT appear in your results.

The first clause of the search (in the main search box) is a MUST - so the results must include whatever goes in that first box. This means that whatever else goes in the search you are essentially starting from a situation where the search in the first box is compulsory. This means that adding a SHOULD clause to this cannot increase the set of results. Adding a SHOULD clause cannot reduce the set of results either (as the clause is optional).

This means that adding a SHOULD clause does not change the total number of results. What it does do is re-order the results when using the relevance ranking, so those items matching the SHOULD clauses come to the top.

Operators, regular expressions and reserved characters

Boolean operators

The familiar boolean operators AND, OR, NOT (alternatively you can write them as &&, ||, !) are supported and can be entered directly into the search box. NOT takes precedence over AND, which takes precendence over OR and you can combine these operators with brackets. These operators can affect the terms to the left and right of them.

You can also use the prefered Elasticsearch operators '+' (which means the term MUST be present) and '-' (which means the term MUST NOT be present). These operators can only affect the terms to the right of them.

The example below finds all results for "shakespeare" without "venice" in.

                shakespeare -venice

By default, any terms you enter in the search box MUST be present. For instance, all of the searches below are identical:

           	shakespeare -venice ode
            	+shakespeare -venice ode
            	+shakespeare -venice +ode

Wildcard operators

Two wildcard operators are available:

                * = Match any character, including no character
                ? = Match any single character; will not match no character

Please avoid wildcard characters occurring at or before the third position as they may produce too many unrelated matches.

Proximity operators

Proximity operators enable you to specify the distance (in number of words) your search terms are from each other in the text. You can specify the number using a tilda and a number after the term itself, to widen the proximity increase the number.

The example below finds king and queen within 5 words or less of each other.

                "king queen"~5

Regular expressions

Searching using regular expressions is supported and terms can be entered directly into the search box wrapped in forward slashes "/". Historical Texts uses Elasticsearch software to index the data and the full list of supported operators for regular expressions can be found on the Elasticsearch website.

The example below shows a regular expression to look for the word William.

                /W[ilam]{0,6}/

Regular expressions support

Support information and tutorials to help you construct regular expressions can be found in a range of places on the web, for example at the Regex One site and Regular Expressions Info.

Reserved characters

Certain characters function as operators in the system and if you want to use them in your search query, but not as an operator, then you will need to escape them with a leading backslash.

The reserved characters are:

                + - && || ! ( ) { } [ ] ^ " ~ * ? : \ /

Fuzzy searching

Fuzzy searching enables you to search for terms that are similar to, but not exactly like our search terms.

"Fuzziness" is measured in terms of the number of edits (insertions and deletions) between one term and another. You can specify the number of edits using a tilda and a number after the term itself.

The example below finds 'king', and all other terms within a single edit of the word 'king':

king~1

The maximum 'fuzziness' supported is 2.
(obviously you can use the 'Fuzzy search' checkbox for fuzzy searching but the operator is also supported in the main search box and allows you to apply the 'fuzziness' to only certain terms within your search)

Variant Forms and Misspelling searching

Users can also choose options for "Variant Forms", "Variant Spellings", "Misspellings" and "Printer Replacements" by clicking the "Fuzzy and Variants" drop-down button and ticking the desired options. All the Variant words that are added to the search will appear below the Search bar, colour-coded by Variant type. You can remove Variant words from your search by clicking the 'x' at the end of the icon.

We recommend that users not use both the 'Fuzzy' and 'Variants' options at the same time, as this may yield unpredictable search results.

Grouping

You can group together terms or search clauses together using parentheses. For example:

shakespeare AND ((merchant AND venice) OR (king AND lear))

Field searching

You can search specific fields by using the field name followed by a colon then the search term. For example

title_all_text:shakespeare

You can use this in combination with other operators. For example:

title_all_text:(shakespeare lear)
title_all_text:lear AND author_all_text:shakespeare~2

You can also search for the presence/absence of fields using '_missing_' and '_exists_'. For example to find all texts without edition information:

_missing_:publication.edition.statement

Bibliographic fields supported in the drop down search box menu:

  • All: _all
  • Title: title_all_text
  • Author: author_all_text
  • Publisher/Printer: lifecycle.publication.publisher.name.value_text
  • Place of publication: lifecycle.publication.place.name.value_text
  • Bibliographic number: bibliographic_number
  • Language: language
  • Subject: subject_all_text
  • Genre/Material type: type_or_genre_all_text
  • Illustration Description: searches both 'physical_descriptors.other' and 'figures'

Also dates:

  • Earliest publication date for text: lifecycle.publication.date.earliest
  • Latest publication date for text: lifecycle.publication.date.latest

Ranges (numeric, date or text)

You can search for numeric, date or text ranges. Inclusive ranges are specified with square brackets [min TO max] and exclusive ranges with curly brackets {min TO max}.

lifecycle.publication.publisher.name.value_text:[aa TO ab]

The example above would find all texts with a word in the publisher that comes between 'aa' and 'ab' in alphabetical order including occurrences of 'aa' and 'ab'. For example it would find the publication statement:

Aat the shop adioyning vnto Saint Mildreds Church in the Pultrye by John Allde

Because of the word 'Aat'

lifecycle.publication.date.earliest:[* TO 1600}

Finds all texts with an earliest publication date before 1600 (up to 1599).

More details on range searching can be found on the Elasticsearch website.

Boosting

Boosting can be used to change the relevance ranking based on specific terms in your search - it won't change the results retrieved, but will change the order. This is done by using the boost operator ^. For example:

shakespeare lear^2

Will boost the importance of 'lear' in the results and so re-order the search set to emphasise those where 'lear' appears more often.

The default boost value is 1, but can be any positive number. A boost value between 0 and 1 reduces relevance of a term.

Advanced search

More search options are available, but are hidden by default. To show the extra options, click the "Advanced Search" drop-down button to the right of the page.

The extra options are:

  • Filter by date - included undated, exclude undated, undated only, or specify a date or range
  • Specify which collection or collections to search on
  • Search on Description only, Full Text Only, or both
  • Filter by Illustrated, Not Illustrated, or both
  • Add rows to introduce additional search terms, using MUST, SHOULD or MUST NOT conditional logic, and searching on any fields available in the normal Search box
  • Reset button to clear down the search interface

Filter by date

By default, searches will return data from all available date ranges. Clicking the drop-down arrow and selecting the radio button next to the date range field allows users to select from "Including Undated", "Excluding Undated" or "Undated Only" options.

"Including Undated" is the default for searching. It also allows for undated records to be included alongside dated records matching a specified range as entered into the date range field (e.g. 1600-1645 plus any undated records matching the other search terms).

"Excluding Undated" will return only dated records matching a specified range as entered into the field (e.g. 1600-1645).

"Undated only" will return only records with no date

Individual years can be entered into the date range field, as can hyphenated ranges (e.g. 1600-1645).

Selecting specific collections will have an effect on the returned results' date range. Users can also use the Selector tool on the Histogram to further refine their search results by date range.

Specify which collection or collections to search on

By default, all collections in the Historical Texts service will be included in a search. To search on a single collection or any custom selection, expand the options for Advanced Search and untick the collections that are not required

Please note selecting specific collections will have a effect on the returned date range of your search results.

Search on Description only, Full Text Only, or both

By default, Historical Texts searches all full text and bibliographical data available. To restrict your search to only the bibliographic data (and not the full text) of the records, select the "Description" radio button. To search only the full text (and not the bibliographic data) select the "Full Text" radio button.

Using the "Full Text" button will also ensure that only results with the full text included will be returned.

Filter by Illustrated, Not Illustrated, or both

To restrict your search to records specifically with or without illustrations, expand the Advanced Search options and click the "Illustrated" or "Not Illustrated" radio button as required.

Add rows to introduce additional search terms

To build up a complex query you can either type your query into the search box directly using Boolean operators or use the "New Row" feature. Simply select "Advanced Search" on the search page to show additional search features. From here you can add new rows by clicking the "New Row" button, and specify whether your terms MUST, SHOULD or MUST NOT appear in your results.

This means that whatever else goes in the search you are essentially starting from a situation where the search in the first box is compulsory. This means that adding a SHOULD clause to this cannot increase the set of results. Adding a SHOULD clause cannot reduce the set of results either (as the clause is optional).

Reset button to clear down the search interface

Should you wish to reset the search interface back to the default values, you can click the red "Reset" button at the bottom of the "Advanced Search" options should you wish. Manually setting the values back to their defaults should work equally well.

Viewer

To view the page images for a text simply select a record from your results list or the browse pages by clicking on the thumbnail or title hyperlink for it. This will open up the viewer, enabling you to view images, full text (where available) and thumbnails side by side. Using the navigation bar at the top of the screen, you can also download individual images (JPEGs) and whole publications (PDFs) and bibliographic citation information (RIS) and get a permanent URL for the publication when viewing an image.

The viewer is comprised of three panels: a central panel displaying the images and a search panel to the left and a details panel to the right which are collapsed by default. To open the left hand panel select the "Search" button on the left hand side of the viewer and to open the right hand panel select the "Details" button on the right of the viewer as shown in the screenshot below.

You can have all three panels open at the same time, or just one or two of them and you can re-size the panels by dragging the handle in the middle of the panel edge (highlighted by red boxes with arrows in the screenshot below) to the left or right. To close either panel click on the cross at the top of the panel edges, highlighted by the small red boxes in the screenshot below.

The options available on each of the three panels are listed below.

Image panel

The central panel in the viewer displays the page images. There is a toolbar on the left hand side of the panel providing the following options:

  • Zoom in
  • Reset zoom (reset the zoom level so the image is the original default size)
  • Zoom out
  • Full screen (opens the page image in full screen mode)
  • Highlight off (feature not available yet but it will hide the search term highlighting)
  • Rotate (rotate image by 90 degrees at a time)

You can use your mouse wheel or trackpad to zoom in and out instead of using the zoom in/out buttons if you prefer and you can drag the image around using your mouse or trackpad.

Search Panel

To open the search panel, select the "Search" button on the left had side of the viewer. You can re-size the panel by dragging the handle in the middle of the panel edge across to the right and you can close the panel using the cross at the top of the panel edge. There are three tabs providing options for navigating publications:

  • Results (this lists images matching your search terms so you can select one to jump straight to that image
  • Thumbnails (this displays thumbnail images for every image in the publication. 30 are loaded at a time and as you scroll down additional thumbnails will continue to load until you reach the end of the publication.
  • Publications (this displays all of the other publications in the platform that match your search terms, so you can quickly jump to a different publication from within the viewer instead of returning to the results page to select another text).

Details panel

To open the details panel click on the "details" button on the right hand side of the viewer. You can re-size the panel by dragging the handle in the middle of the panel edge across to the left and you can close the panel using the cross at the top of the panel edge. There are two tabs providing additional information on the publication:

  • Details (this displays the bibliographic information for a publication)
  • Full text (this displays the full text supporting the images, where available).

Please note that full text is not available for all publications in Historical Texts. There is full text available for every title in ECCO and the BL 19th Century collections and this has been machine generated using Optical Character Recognition (OCR) software. There is full text for around a third of the EEBO collection currently and this has been manually keyed by the Text Creation Partnership. For the rest of the EEBO collection there are images and bibliographic information only. When full text is not available the full text tab will display the message "Full text not available".

Downloading

When you have opened a publication in the viewer, choose the "download" button and select from a range of download options available:

  • individual image currently open in the viewer as a jpeg file
  • whole publication as a PDF file
  • citation information in RIS format to import into reference management software

All of these options provide an instant download of either a jpeg, PDF or RIS format file. These files will either open or download in your browser, depending on the browser you use and your local configuration settings for it.

Please note: the download button is only enabled on the navigation bar when you have opened a publication in the viewer. On all other pages it is currently disabled and greyed out with a message stating "feature not available" as you can only download a publication when you have selected and opened it.

Stable URLs

The "Share" button on the navigation bar at the top of the screen provides a stable (permanent) URL for the publication you are viewing for you to copy and paste into your documents or web pages. The button is deactivated and greyed out until you open a publication in the viewer, at which point it is enabled and when you select it a dialogue box containing a stable URL for the publication will be displayed.

Saving searches and publications

You can save searches and publications for future sessions in Historical Texts.

  • Save searches: A green "save search" button enables you to save your current search. Your saved search can be viewed and run from the "saved searches" option under "My Texts" on the toolbar at the top of the screen.
  • Save publications: You can save publications to a list for future reference by selecting the checkbox next to one or more results from your results list. The publications will be added to the "My Saved Items" tab and from there you can add them to a new list or an existing list. You can also access your saved items from "My Texts" on the toolbar at the top of the screen.

You can create up to 99 searches and lists as well as save up to 99 publications in each list.

Personalisation options are only available when logging in via an institutional account (UK Federation Shibboleth/OpenAthens). If you access Historical Texts directly (ip authentication) then you will be prompted to login with an institutional account when you click on "save search", "My Saved Items" or an option under "My Texts".
