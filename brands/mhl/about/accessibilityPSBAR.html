Accessibility information for UK Medical Heritage Library Collection on Historical Texts Labs

This accessibility statement applies to https://ukmhl.historicaltexts.jisc.ac.uk.

This website is run by Jisc. We want as many people as possible to be able to use this website. For example, that means you should be able to:

  • Change colours, contrast levels and fonts using browser plugins
  • Zoom in up to 200% without the text spilling off the screen.
  • Navigate most of the website using just a keyboard
  • Listen to most of the website using a screen reader (tested using NVDA)
  • Skip to main content on all but one page

We’ve also made the website text as simple as possible to understand.

AbilityNet has advice on making your device easier to use if you have a disability.

How accessible this website is

We know some parts of this website are not fully accessible:

  • Although we have made recent improvements to the site, the viewer still has some issues. If you are having difficulties, the best solution may be to open the item in the viewer and download the PDF/text files from the download menu. This can be operated by keyboard and even when styling is turned off for the viewer page, the buttons still operate. This would also be our recommendation if you need to use the viewer at a magnification above 200%.
  • Quite a few of the visualisation tools are not operable by keyboard. The following tools can be used by keyboard only: NGram Viewer, Wall of Images and, with some caveats, the Dendrogram
  • It is not possible to operate the histogram using the keyboard but the same result can be achieved using the advanced search date filter, which can be operated by keyboard.
  • Please note that certain drop down menus in the tool bar and advanced search can only be operated by tab rather than arrow keys.
  • The site is not fully usable in the portrait orientation on a small screen device. We recommend viewing the site in landscape orientation if possible. If this is not possible, we recommend viewing search results in the portrait orientation and then downloading the PDF from the viewer rather than trying to operate the viewer in portrait orientation.
  • When you view some of our pages at or above 200% zoom, although the text reflows, parts of some images and videos are lost off the right-hand side of the page. Some of these images can be viewed as full page images (e.g. in the Quick Reference Guide) so check if they can be clicked on to enlarge them that way. An alternative source for our support videos is the Historical Texts YouTube channel.
  • We have very few PDFs on the UK Medical Heritage Library Site. The Quick Reference Guide PDF is also available in HTML format.
  • Please note that is also possible to access all the content of the UK Medical Heritage Library via the main Historical Texts platform even if you are not a subscriber.

Feedback and contact information

If you need information on this website in a different format like accessible PDF, large print, easy read, audio recording or braille:

  • email historicaltexts@jisc.ac.uk

We'll consider your request and get back to you.

Reporting accessibility problems with this website

We're always looking to improve the accessibility of this website. If you find any problems not listed on this page or think we’re not meeting accessibility requirements, contact the helpdesk (historicaltexts.jisc.ac.uk).

Enforcement procedure

The Equality and Human Rights Commission (EHRC) is responsible for enforcing the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018 (the 'accessibility regulations'). If you’re not happy with how we respond to your complaint, contact the Equality Advisory and Support Service (EASS).

Technical information about this website's accessibility

Jisc is committed to making its website accessible, in accordance with the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018.

Compliance status

This website is partially compliant with the Web Content Accessibility Guidelines version 2.1 AA standard, due to the non-compliances and exemptions listed below.

Non-accessible content

The content listed below is non-accessible for the following reasons.

Non-compliance with the accessibility regulations

A small number of our pages lack headings which makes it harder for those using screen readers to navigate. This fails WCAG 2.1. success criterion 1.3.1 (Info and Relationships). We plan to ensure that all pages in the new site we are designing have appropriately used headings on all pages.

If styling is removed from the viewer page, the content is no longer fully usable. This fails WCAG 2.1 success criterion 1.3.2 (Meaningful sequence). The best solution to this is to download the PDF/text and read it outside the viewer. This operation can also be carried out from the page with the styling turned off.

The site was originally designed with only desktop use in mind, and although improvements have been made to increase the usability on smaller screen tablet/mobile devices, it is still not really possible to use the site properly in a portrait orientation. This fails WCAG 2.1 success criterion 1.3.4 (Orientation). We plan to ensure that the new site we are designing is usable in both portrait and landscape orientation from the start.

When the site is zoomed in to 200% or more, although text reflows as required, some images and videos "fall off" the right-hand side of the page rather than resizing. When using the viewer at a magnification of above 200% it becomes very difficult to use the content of the side panels and some of this is lost. These fail WCAG criterion 1.4.10 (Reflow). Some workarounds for this issue are listed earlier in the document. We plan to use a fully fluid layout for the new site we are designing which will eliminate these issues.

A number of the visualisations and the medical events on the timeline cannot be operated by keyboard which fails WCAG criterion 2.1.1 (Keyboard). We have made the visualisations as accessible by keyboard as possible on the current platform. The dendrogram tabbing can behave in an unexpected way, appearing to move backwards. This is due to how the nodes are created on the fly. As the visualisations were created by a third party, we are unable to easily change this. We plan to investigate designing new visualisations which will be more accessible as we integrate the UK Medical Heritage content into the new platform we are building. Please note that the zoom in and zoom out keys in the viewer cannot be operated by keyboard when using the viewer in Chrome. A workaround for this is to use the viewer in FireFox which does not suffer from this issue. It is also not possible to operate the histogram using the keyboard but the same result can be achieved using the advanced search date filter, which can be operated by keyboard.

A "Skip to main content," link is not currently available in the viewer, and some of the visualisation pages which fails WCAG criterion 2.4.1. (Bypass Blocks). These have been added to all other pages, but technical difficulties prevent it from being added to the third party viewer and visualisation pages. We plan to ensure that all pages in the new platform have bypass blocks in place.

Most pages have a unique title but currently the search results page and browse pages have the identical title of "results." This makes distinguishing these pages for those using a screen reader more difficult and fails WCAG 2.1 criterion 2.4.2 (Page titled). We plan to ensure that all pages have unique titles in the new platform we are building.

On some pages, there are links with the same text that go to different places. This can be confusing for screen reader users and fails WCAG criterion 2.4.4 (Link Purpose In Context). We plan to differentiate these links either by updating the text or using aria-labelledby by 23 September 2020. For those where this is not possible due to the structure of the site, we will change we way we create menus in the new site we are designing.

We are aware that although improvements have been made to the navigability of the site by keyboard/screen reader, it is still sometimes difficult to see where you are which fails WCAG criterion 2.4.7 (Focus visible). We are also aware of at least one automated fail for this criterion. We plan to use a more consistent design pattern which will make focus states more perceivable in the new site we are designing.

Disproportionate burden

We are not claiming a disproportionate burden for any area of improvements that are required.

Content that's not within the scope of the accessibility regulations

PDFs and other documents

The site only uses one PDF document (Quick Reference Guide) but this information is also available in HTML format.

The accessibility regulations do not require us to fix PDFs or other documents published before 23 September 2018 if they’re not essential to providing our services. Any new PDFs or Word documents we publish will meet accessibility standards or the information will be provided in a more accessible HTML format.

What we're doing to improve accessibility

We plan to merge some features of the UK Medical Heritage Library site into a new platform we are building for Historical Texts over the next three years, including a package of work on redesigning the visualisations to make them more accessible and moving to the use of the Universal Viewer.

Preparation of this accessibility statement

This statement was prepared on 8 September 2020. It was last reviewed on 15 September 2020.

This website was last tested on 15 September 2020. The test was carried out members of the Historical Texts team.

We selected several pages that represented different types of content on the site ensuring there were images, links and forms across the selection. We tested:

  • UK Medical Heritage Library home page
  • Search results page
  • Viewer
  • Support home page
  • Quick Reference guide page
  • About home page
  • Browse page
  • A selection of the visualisation pages

We used a mixture of manual testing (including using NVDA) and automated testing using both WAVE and SiteImprove.
